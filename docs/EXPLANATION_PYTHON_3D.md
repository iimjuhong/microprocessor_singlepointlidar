# 파이썬 실시간 3D 코드 완전 분석 (arduino_3d.py)

이 문서는 아두이노가 초당 수백 개씩 쏟아내는 직렬 통신 데이터를 받아 실시간으로 3D 렌더링을 수행하는 파이썬 코드(`arduino_3d.py`)를 약 500줄 전체에 걸쳐 상세하게 해부한 가이드입니다.
멀티스레딩, 칼만 필터, 비동기 큐, 3D 기하학 변환 등 수많은 고급 기술이 총망라되어 있습니다.

---

## 1. 라이브러리 및 데이터 구조체 (Lines 1 ~ 44)

```python
import queue, threading, serial
import matplotlib.pyplot as plt
import numpy as np
```
*   가장 핵심이 되는 라이브러리들입니다. `threading`과 `queue`로 아두이노 데이터가 밀리지 않게 병렬로 받고, `serial`로 통신하며, `numpy`로 수만 개의 좌표를 한 번에 연산해 `matplotlib`으로 화면에 그립니다.

### `@dataclass ScanPoint`
*   점 1개의 정보를 담는 가벼운 그릇입니다.
*   `yaw_deg`(좌우 각도), `pitch_deg`(상하 각도), `distance_cm`(레이저 거리), `strength`(반사 강도) 변수를 가집니다.

### `@dataclass FilterConfig`
*   3D 변환을 할 때 필요한 설정값들입니다.
*   `center_offset_cm`: 스캐너 렌즈가 회전축 정중앙이 아니라 앞쪽으로 튀어나와 있어서 생기는 왜곡을 잡아주기 위한 거리 값입니다.
*   `min_distance_cm, max_distance_cm`: 거리가 너무 가깝거나(센서 사각지대) 너무 먼 쓰레기 데이터들을 무시하기 위한 기준입니다.

## 2. EKF (확장 칼만 필터) 구현부 (Lines 46 ~ 91)
아두이노 모터 기어의 백래시(헐거움)로 인해 측정된 각도가 덜덜 떨리는 현상을 소프트웨어적으로 완벽하게 잠재우는 클래스입니다.

### `YawKalmanFilter` 클래스
*   **상태 모델 (`self.x`)**: 2x1 행렬로 `[현재 각도, 현재 각속도]`를 추적합니다.
*   **시스템 노이즈 (`self.Q`)**: "바람이나 기계적 마찰 때문에 모터 속도가 살짝은 변할 수 있다"고 가정하는 허용 오차입니다.
*   **측정 노이즈 (`self.R`)**: "엔코더 센서 값 자체를 100% 믿을 수는 없다"고 가정하는 센서 오차율입니다.
*   **`update(time_s, meas_yaw)` 함수**: 
    1.  **시간 차(dt) 계산**: 저번 데이터 이후로 시간이 얼마나 흘렀는지 계산합니다.
    2.  **Wrap-around 처리**: 각도가 359도에서 0도로 팍 넘어갈 때, 필터가 "어? 모터가 갑자기 뒤로 359도 돌았네?" 라고 착각하지 않도록 360도를 더해주거나 빼주어 부드럽게 이어줍니다.
    3.  **예측 (Predict)**: 속도 x 시간(dt)을 더해 모터가 지금쯤 어디 있어야 하는지 수학적으로 예측합니다.
    4.  **업데이트 (Update)**: 센서가 측정한 실제 값과 내가 예측한 값을 섞습니다. 칼만 이득(`K`)이라는 마법의 비율을 써서, 두 값 중 더 믿을만한 쪽으로 가중 평균을 냅니다. 결과적으로 **기어 떨림이 완전히 사라진 매끄러운 각도**가 탄생합니다.

## 3. 멀티스레드 시리얼 수신기 (Lines 94 ~ 179)
메인 프로그램이 3D 그래픽을 그리느라 바빠도 데이터를 잃어버리지 않게 백그라운드에서 데이터를 받아주는 클래스입니다.

### `LidarClient` 클래스
*   `__init__()`: `serial.Serial`을 열고 데이터를 담을 `self.points` 리스트와 프린트 메시지를 담을 `queue.Queue()`를 만듭니다. 그리고 `_reader` 함수를 **Daemon Thread**(메인 프로그램이 꺼지면 같이 꺼지는 닌자 스레드)로 실행시킵니다.
*   **`_reader()` (가장 중요)**: 
    *   무한 루프(`while self._running:`)를 돌며 아두이노가 보내는 줄을 읽습니다 (`readline()`).
    *   `SYNC`로 시작하는 줄이면 쉼표로 변수를 자릅니다.
    *   아두이노가 보낸 생 카운트 `enc_cnt`를 660으로 나누어 `raw_yaw_deg`로 만듭니다.
    *   위에서 만든 칼만 필터 인스턴스(`kf.update`)에 넣어서 덜덜거리는 각도를 부드럽게 폅니다.
    *   완성된 `ScanPoint`를 리스트에 넣습니다. 이때 `with self._lock:` 을 써서, 리스트에 점을 넣는 순간 메인 스레드가 리스트를 빼앗아 가지 못하게 자물쇠를 채웁니다 (Thread-Safe).

*(그 아래의 `SimulatedClient`는 아두이노 없이도 파이썬 코드가 잘 도는지 테스트하기 위해 가짜 원통 데이터를 빙글빙글 만들어내는 목업(Mock-up) 클래스입니다.)*

## 4. 3D 직교 좌표 변환부 (Lines 241 ~ 290)
레이저가 쏜 극좌표 `(거리, 각도)`를 화면에 그릴 `(X, Y, Z)`로 바꾸는 수학 계산입니다.

### `point_to_xyz(point, config)`
1.  **필터링**: 점의 거리가 10cm 미만이거나 강도가 너무 낮으면 아예 무시(`return None`)합니다.
2.  **각도 단위 변환**: 삼각함수를 쓰기 위해 도(Degree)를 라디안(Radian)으로 바꿉니다.
3.  **높이 및 빗변 계산**: 
    *   `Z = 거리 * sin(pitch)` (위로 얼마나 들렸나)
    *   `r_horizontal = 거리 * cos(pitch)` (바닥에 내린 빗변의 길이)
4.  **오프셋 보정 (핵심 기하학)**:
    *   레이저 센서가 회전축 정중앙에 있지 않습니다. `center_offset_cm`만큼 앞에 튀어나와 있습니다.
    *   따라서 진짜 물체의 반경(`actual_radius`)은 `중심축에서 렌즈까지의 거리 - 렌즈에서 물체까지의 빗변`이 됩니다. 이 연산이 없으면 스캔된 물체의 배가 불룩하게 찌그러집니다.
5.  **X, Y 계산**: `X = 실제반경 * cos(yaw)`, `Y = 실제반경 * sin(yaw)`

### `polar_to_xyz(points)`
위의 함수를 활용해 수만 개의 `ScanPoint` 리스트를 거대한 Numpy 2차원 배열 `(N, 4)`로 한 방에 바꿔주는 도우미 함수입니다. Numpy 배열로 만들어야 렌더링이 빠릅니다.

## 5. 유틸리티 함수들 (Lines 292 ~ 363)
*   **`save_csv`**: 파이썬 프로그램을 끄기 직전에, 모아둔 모든 점들을 `csv.writer`를 써서 엑셀 파일로 쭉 써 내려갑니다.
*   **`choose_port`**: 실행할 때 포트를 지정하지 않으면, 꽂혀있는 USB 포트 목록을 쫙 보여주고 사용자가 키보드로 고르게 해주는 친절한 함수입니다.
*   **`color_values`**: 3D 화면에서 점에 색깔을 칠할 때, 기준을 높이(`z`)로 할지, 중심 거리(`distance`)로 할지, 센서 반사 강도(`strength`)로 할지 골라서 Numpy 배열 1줄짜리로 빼줍니다.

## 6. 키보드 입력 스레드 (Lines 366 ~ 383)
### `input_worker(client, stop_event)`
시각화 화면이 돌아가는 동안 터미널 창에서 사용자의 키보드 입력을 감시합니다.
*   명령을 치면 `client.send()`로 아두이노에게 보냅니다.
*   `q`를 치면 아두이노에게 원점으로 돌아가라는 `z` 명령을 보내고, 3초를 센 다음 시스템 종료 플래그(`stop_event.set()`)를 발동시킵니다.

## 7. 메인 렌더링 루프 (Lines 385 ~ 488)

### `main()` 구동 순서
1.  **인자 파싱 (Argparse)**: 터미널에서 `python arduino_3d.py --port COM5 --color-by strength` 처럼 친 옵션들을 읽어 들입니다.
2.  **객체 생성**: 통신(`LidarClient`), 키보드 입력 스레드, 화면 창(`plt.figure`)을 띄웁니다.
3.  **인터랙티브 모드 (`plt.ion()`)**: 코드가 도는 동안 사용자가 마우스로 3D 창을 이리저리 돌려볼 수 있게 해줍니다.

### `while` 메인 루프
화면을 업데이트하는 심장부입니다.
*   `points = client.snapshot()`: 백그라운드 스레드가 그동안 모아둔 점들을 싹 가져옵니다.
*   **개수 비교 (`len(points) != last_count`)**: 점 개수가 달라졌을 때만 화면을 그립니다. 계속 그리면 CPU가 터져버리기 때문입니다.
*   **그리기 (`ax.scatter`)**: `ax.cla()`로 이전 그림을 다 지우고, 아까 만든 넘파이 배열(`xyz`)을 통째로 쑤셔 넣어 3D 구름을 만듭니다. 
*   **비율 고정 (`ax.set_xlim, ylim, zlim`)**: 점이 늘어날 때마다 화면이 줌인/줌아웃 되며 울렁거리지 않도록, 가장 큰 점을 기준으로 카메라 렌즈 화각(Limit)을 꽉 잡아둡니다.
*   `plt.pause(0.05)`: CPU가 잠깐 쉬면서 Matplotlib이 그래픽 카드를 써서 화면을 새로고침 할 시간을 0.05초 줍니다.

### `finally` 구문
사용자가 창의 X 버튼을 누르거나 `q`를 쳐서 루프가 깨지면 무조건 실행되는 안전장치입니다.
*   지금까지 메모리에 있던 모든 점들을 `polar_to_xyz`로 변환하고 `save_csv`를 불러 파일로 영구 저장합니다.
*   `client.close()`로 백그라운드 스레드를 깔끔하게 죽이고 포트를 닫습니다.

이 코드는 실시간 통신의 병목(Bottleneck) 현상을 막기 위해 큐와 스레딩을 극한으로 활용하고, 거기에 칼만 필터까지 얹어 스캐너의 기계적 한계를 소프트웨어로 완벽히 극복해 낸 역작입니다.
