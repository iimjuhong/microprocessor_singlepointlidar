# 아두이노 코드 완전 분석 (arduino_lidar_scanner_register.ino)

이 문서는 총 850줄이 넘는 아두이노 코드를 처음부터 끝까지, 변수 하나와 함수 하나까지 빼놓지 않고 아주 상세하게 해부한 완벽 가이드입니다. 
아두이노의 하드웨어 레지스터(Register)를 직접 건드리는 고급 테크닉이 사용되었으므로, 각 줄이 왜 이렇게 작성되었는지 꼼꼼하게 설명합니다.

---

## 1. 헤더 포함 및 핀(Pin) 정의 (Lines 1 ~ 50)

```cpp
#include <SoftwareSerial.h>
```
*   아두이노 우노(ATmega328P)는 하드웨어 시리얼 포트(UART)가 딱 1개뿐입니다. 이 1개는 컴퓨터(파이썬)와 통신하는 데 써야 하므로, LiDAR 센서(TFmini)와 통신하기 위해 소프트웨어 시리얼 라이브러리를 하나 가져옵니다.

### 레지스터 핀맵 정의
*   **엔코더 핀 (`PD3`, `PD2`)**: `PD3`은 아두이노의 D3핀이며 외부 인터럽트 `INT1`이 연결된 핀입니다. `PD2`는 D2핀입니다.
*   **DC 모터 핀 (`PD4`, `PD5`, `PD6`)**: L298N 모터 드라이버와 연결됩니다. 특히 `PD6`(D6 핀)은 타이머0의 `OC0A` 핀과 물리적으로 연결되어 있어 레지스터 조작으로 하드웨어 PWM을 출력할 수 있습니다.
*   **스텝 모터 핀 (`PB3`, `PB2`, `PB5`, `PB4`)**: ULN2003 드라이버와 연결됩니다. 아두이노의 10, 11, 12, 13번 핀이며 전부 PORTB(B포트)에 속해 있어 한 번의 레지스터 조작(`PORTB = ...`)으로 4개 핀을 동시에 켜고 끌 수 있습니다.
*   **`STP_MASK`**: 스텝 모터가 쓰는 핀들만 1로 켜둔 비트 마스크입니다. 다른 핀을 건드리지 않고 스텝 모터 핀만 덮어씌울 때 씁니다.

## 2. 스캔 파라미터 및 PID 설정 (Lines 52 ~ 112)

### 기하학 및 스캔 상수
*   `TARGET_DISTANCE_CM = 48.0`, `TARGET_VERTICAL_RES_CM = 0.5`: 스캐너가 물체를 스캔할 때 0.5cm씩 일정한 간격으로 위로 올라가게 하기 위한 기준 거리입니다.
*   `YAW_REVS_PER_PITCH = 1`: 층을 바꾸기 전에 모터가 제자리에서 몇 바퀴를 돌지 결정합니다. (1바퀴)
*   `ENCODER_COUNTS_PER_REV = 660.0`: 실측 보정을 통해 얻은 터닝테이블 1회전 당 엔코더 카운트 수입니다. (과거 1320에서 보정됨)
*   `WARMUP_COUNTS = 0`: 과거에는 워밍업으로 0.5바퀴를 날렸으나, 현재는 1바퀴 스캔이므로 버리는 데이터 없이 0으로 설정합니다.
*   `MAX_TARGET_HEIGHT_CM = 48.0`: 스캔 종료 최대 높이입니다. 센서 거리 48cm와 높이 48cm가 같아지므로 최대 스캔 각도(Pitch)는 45도가 됩니다.

### PID 속도 제어 변수
*   `Kp = 0.6, Ki = 0.15, Kd = 0.02`: 속도를 제어하는 마법의 숫자들입니다.
*   `TARGET_SPEED = 200.0`: 초당 200카운트의 속도로 회전하라는 뜻입니다.
*   `speedIntegral, speedPrevSpeed, speedFilteredDerivative`: 적분값 누적, 미분 킥 방지, 지수이동평균(EMA) 필터를 위해 이전 계산값들을 계속 저장해 두는 변수들입니다.

### 스텝 모터 하프스텝 시퀀스 (`HALF_STEP_SEQ`)
*   크기가 8인 배열입니다. 28BYJ-48 스텝 모터를 부드럽게(하프스텝) 돌리기 위해 코일에 전기를 넣는 순서 8단계를 비트(Bit) 형태로 미리 계산해 둔 표입니다.

## 3. 전역 상태 변수들 (Lines 114 ~ 144)

*   `SoftwareSerial tfmini(9, 8)`: LiDAR 통신 객체입니다.
*   `volatile long encoderCount`: 인터럽트 안에서 실시간으로 변하는 모터의 현재 위치입니다. (반드시 `volatile`을 붙여야 CPU가 최적화 핑계로 무시하지 않습니다)
*   `volatile bool targetRevsReached`: 목표한 바퀴 수(예: 2바퀴)를 다 돌았으면 `true`가 되는 깃발(Flag)입니다.
*   `ScannerState state`: 스캐너의 현재 상태(`IDLE` 대기, `SPINNING_YAW` 회전중, `MOVING_PITCH` 고개드는중, `FINISHED` 종료)를 저장합니다.

## 4. UART (시리얼 통신) 직접 제어 (Lines 146 ~ 218)
무겁고 느린 `Serial` 클래스를 버리고 레지스터를 직접 타격합니다.

*   **`uart_init_115200()`**:
    *   `UCSR0A`의 `U2X0` 비트를 켜서 배속 모드를 만듭니다.
    *   `UBRR0L = 16`: 16MHz 클럭에서 115200 통신 속도를 맞추기 위한 마법의 숫자(Baud rate divisor)입니다.
    *   `UCSR0B`: `RXEN0`(수신 켜기), `TXEN0`(송신 켜기)를 설정합니다.
*   **`uart_putc(char c)`**: `UCSR0A` 레지스터의 `UDRE0`(송신 버퍼가 비었음) 비트가 1이 될 때까지 `while`문으로 기다렸다가, 비면 `UDR0` 레지스터에 1바이트를 꽂아 넣습니다.
*   **`uart_puts_P()`**: 아두이노의 귀한 RAM을 아끼기 위해 문자열을 플래시 메모리(PROGMEM)에 저장해두고, `pgm_read_byte`로 한 글자씩 꺼내서 전송하는 고급 기법입니다.
*   **`uart_print_int()`, `uart_print_float()`**: C 표준 라이브러리인 `ltoa`와 `dtostrf`를 써서 숫자를 문자로 바꾼 뒤 전송합니다.

## 5. GPIO 및 모터 제어 (Lines 220 ~ 286)

### `setup_gpio()`
*   `DDRD`, `DDRB` (Data Direction Register): 0을 넣으면 입력 핀, 1을 넣으면 출력 핀이 됩니다.
*   엔코더 핀은 입력으로, 모터 핀들은 전부 출력으로 설정합니다. 엔코더 핀에는 `PORTD |= ...`를 통해 칩 내부의 풀업 저항(Pull-up resistor)을 켭니다.

### `setDcMotorPwm_raw(uint8_t duty)`
*   `analogWrite(6, duty)`를 완전히 대체하는 함수입니다.
*   타이머0은 시스템이 이미 Fast PWM 모드로 켜놓은 상태입니다.
*   `duty`가 0이면 핀 연결(`COM0A1` 비트)을 끊어서 완전히 LOW를 만듭니다. 0이 아니면 핀을 타이머와 연결하고 `OCR0A` 레지스터에 0~255 값을 꽂아 하드웨어 PWM을 즉시 내보냅니다.

### `setDcMotor(int pwm)`
*   부호에 따라 방향 핀(`IN1`, `IN2`)을 HIGH/LOW로 설정하고 절댓값을 PWM으로 넘겨줍니다.

## 6. 인터럽트 (엔코더 & 스텝퍼) (Lines 288 ~ 412)

### 엔코더 외부 인터럽트 (`setup_encoder_interrupt`, `ISR(INT1_vect)`)
*   `EICRA` 레지스터를 `CHANGE` 모드로 설정해 핀 상태가 오르내릴 때마다 쳐다보게 합니다.
*   `EIMSK` 레지스터로 `INT1`의 스위치를 켭니다.
*   **`ISR(INT1_vect)`**: 
    1. `PIND` 레지스터를 한 번에 통째로 읽어서(매우 빠름) A핀과 B핀의 상태를 추출합니다.
    2. A와 B가 같으면 정방향(+1), 다르면 역방향(-1)으로 카운트를 계산합니다.
    3. 만약 지금 카운트가 목표치(`encoderThreshold`)를 넘었다면, 지체 없이 `targetRevsReached = true`를 띄우고 그 순간의 정확한 카운트(`encoderCountAtTrigger`)를 백업합니다.

### 스텝퍼 타이머 인터럽트 (`setup_stepper_timer`, `ISR(TIMER1_COMPA_vect)`)
*   타이머1을 1024 프리스케일러, **CTC 모드(Clear Timer on Compare Match)** 로 세팅합니다.
*   `OCR1A = 520`: 약 초당 30번 인터럽트가 걸리게 맞춘 값입니다.
*   **`ISR(TIMER1_COMPA_vect)`**: 
    초당 30번 자동으로 실행되면서, 지금 위치(`stepPosition`)가 목표(`stepTarget`)보다 작으면 하프스텝 배열 인덱스를 올리고, 크면 내립니다. 그리고 `PORTB` 레지스터의 하위 비트에 배열 값을 엎어치기 해서 핀 4개를 동시에 조작합니다. 메인 루프에 딜레이가 걸려도 모터는 완벽히 돌아갑니다.

## 7. 스캔 로직 (Lines 488 ~ 640)

### `updateMeasurement()` (층 올리기)
*   엔코더 인터럽트가 "다 돌았습니다!" 하고 깃발(`targetRevsReached`)을 흔들면 실행됩니다.
*   즉시 `setDcMotor(0)`으로 모터를 세웁니다.
*   **오버슛 계산**: 브레이크를 잡아도 모터는 관성 때문에 조금 더 밀려납니다. 인터럽트가 트리거된 시점과 실제 목표 카운트의 차이를 계산해, 다음 층을 시작할 때 카운트를 0이 아니라 밀려난 만큼 앞당겨서 시작하게 합니다. (완벽한 동심원 유지)
*   **아크탄젠트 각도 계산**: `currentPitchLevel`을 올리고, `atan2(높이, 48.0cm)`를 계산해 스텝 모터가 몇 도를 들어야 0.5cm가 올라가는지 계산합니다.

### `updateSpeedPID()` (심화 제어기)
*   `millis()`를 확인해 20ms마다 한 번씩 돕니다.
*   현재 속도 = (지금 카운트 - 20ms 전 카운트) / 0.02초
*   **적분 (I)**: 오차를 계속 더하다가 범위를 넘으면 자릅니다 (Anti-Windup).
*   **미분 (D)**: 목표 속도가 갑자기 바뀔 때 차가 튀어나가는 현상(Derivative Kick)을 막기 위해, 에러의 변화량이 아닌 **현재 속도의 변화량**을 미분합니다. 그리고 노이즈에 강하게 만들기 위해 EMA 필터(`alpha = 0.3`)로 부드럽게 깎습니다.
*   **Feedforward**: 200속도를 내기 위해 기본적으로 깔아줘야 하는 PWM을 미리 더해주어, PID가 0부터 고생하지 않게 만듭니다.

### `startYawSpin()`, `beginScan()`, `haltScan()`
*   스캐너의 상태를 바꾸고, PID 변수들을 0으로 초기화하며, 스캔을 시작하거나 정지시키는 헬퍼 함수들입니다.

## 8. 메인 루틴 (Lines 722 ~ 857)

### `handleCommand(char cmd)`
파이썬에서 문자가 오면 처리합니다.
*   `s`: 스캔 시작 (`beginScan`)
*   `h`: 정지 (`haltScan`)
*   `z`: 스텝 모터 고개 원위치 (0도)
*   `t`: 시스템 상태 디버그 출력

### `setup()`
지금까지 설명한 모든 레지스터 초기화 함수들을 하나씩 부르고, `sei()`로 글로벌 인터럽트를 켜줍니다.

### `loop()`
1.  **명령 수신**: `uart_available()`로 문자가 왔는지 확인하고 읽습니다.
2.  **센서 읽기**: `readTfminiFrame()`으로 LiDAR 거리를 업데이트합니다.
3.  **데이터 출력**: 스캐너가 도는 중(`SPINNING_YAW`)이고 새 LiDAR 데이터가 왔다면, **`WARMUP_COUNTS`를 넘었는지 꼭 확인**합니다. 넘었다면 `SYNC`라는 글자와 함께 시간, 위치, 거리를 쉼표로 이어 붙여 파이썬으로 쏩니다.
4.  **상태 전이**:
    *   만약 스텝 모터가 고개를 다 들었다면(`stepper_distanceToGo() == 0`), 흔들림이 멎게 0.2초 기다렸다가 DC 모터를 켭니다(`startYawSpin`).
    *   DC 모터가 도는 중이라면 바퀴 수를 다 채웠는지 확인(`updateMeasurement`)하고, PID로 속도를 맞춥니다(`updateSpeedPID`).

이렇게 850줄의 코드가 유기적으로 맞물려, 16MHz짜리 장난감 칩 하나로 로봇 공학 수준의 리얼타임 하드웨어 제어를 이뤄내고 있습니다.
